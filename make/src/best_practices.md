# 最佳实践

## 风格规范

在编写代码时，遵循一致的风格规范可以提高代码的可读性和维护性。推荐使用以下规范：

- **命名约定**：变量和函数使用小写字母和下划线分隔（如 `my_variable`），类名使用驼峰命名（如 `MyClass`）。
- **缩进**：使用4个空格进行缩进，避免使用制表符。
- **注释**：为复杂逻辑添加注释，保持注释简洁明了。
- **行长度**：每行代码不超过80个字符，便于阅读。

## 目录组织

良好的目录结构有助于项目的可扩展性和协作。建议的组织方式如下：

- **根目录**：包含 `README.md`、`LICENSE` 和配置文件。
- **源码目录**：如 `src/` 或 `lib/`，存放主要代码文件。
- **测试目录**：如 `tests/`，存放单元测试和集成测试。
- **文档目录**：如 `docs/`，存放项目文档和API说明。
- **资源目录**：如 `assets/` 或 `static/`，存放图片、样式等静态资源。

## 常见陷阱

避免这些常见错误可以减少bug和维护成本：

- **全局变量滥用**：尽量避免使用全局变量，以防意外修改导致的副作用。
- **硬编码值**：使用常量或配置文件替代硬编码值，提高灵活性。
- **忽略错误处理**：始终添加适当的异常处理，避免程序崩溃。
- **过度耦合**：模块间保持低耦合，通过接口或抽象层进行交互。
- **缺乏测试**：编写单元测试覆盖关键功能，确保代码质量。
