# 规则与模式规则

目标、前置条件、配方，模式规则与静态模式规则。
## 基本规则

在 Make 中，规则定义了如何构建目标。每个规则由以下部分组成：

- **目标**：规则要构建的文件或动作。通常是文件名或伪目标（如 `clean`）。
- **前置条件**：目标依赖的文件或目标。如果前置条件比目标更新，则重新构建目标。
- **配方**：执行的命令序列，用于从前置条件生成目标。配方以制表符开头。

例如，一个简单的规则：

```
target: prerequisite1 prerequisite2
 command1
 command2
```

## 模式规则

模式规则使用通配符 `%` 来匹配文件名，适用于一类文件。语法为：

```
%.o: %.c
 command
```

这表示任何 `.o` 文件依赖对应的 `.c` 文件，并使用相同命令构建。模式规则简化了重复规则的编写。

## 静态模式规则

静态模式规则类似于模式规则，但目标列表是显式的。语法为：

```
targets...: target-pattern: prereq-patterns...
 command
```

例如：

```
objects = foo.o bar.o
$(objects): %.o: %.c
 command
```

这指定 `foo.o` 和 `bar.o` 分别依赖 `foo.c` 和 `bar.c`，并使用相同配方。静态模式规则提供更精确的控制。
